# Specialist Overview  
The full-stack delivery specialist supports reliable and secure software delivery by converting feature, defect, quality, or release requests into bounded implementation guidance, delivery-ready plans, and evidence-backed test and release strategies. It focuses on application-layer delivery across frontend, backend, API, and data flow, without assuming ownership of infrastructure, governance, or approval authority.

## Who uses this  
- Software engineer  
- Technical lead  
- Delivery lead  
- Code reviewer  
- Quality reviewer  
- Product engineering stakeholder  

## What it produces  
- Service or support disposition with scope verdict, evidence used, confidence level, and resolution state  
- Implementation plan with boundaries, dependencies, owners, sequence, assumptions, and acceptance gates  
- Review findings with severity, risk tags, evidence references, affected artifacts, and remediation guidance  
- Test matrix and release readiness checklist with coverage intent, pass criteria, blockers, rollback triggers, and monitoring checks  
- Stakeholder communication summary detailing completed work, unresolved items, assumptions, and next owner  
- Escalation packet with missing facts, conflicting authority, confidentiality limits, handoff target, and safe interim guidance  
- Out-of-scope and orchestrator-return notes when work leaves the full-stack delivery lane  

## Validation  
- 18 evaluation scenarios, pass rate 1  
- Source coverage includes National Institute of Standards and Technology, OWASP Foundation, Association for Computing Machinery, and Ken Schwaber and Jeff Sutherland  
- Human verification and functional audit completed  

## Limitations  
- Requires clear input on feature, defect, quality, or release requests to generate accurate outputs  
- Requires access to API contracts, code constraints, and release expectations  
- Requires defined boundaries for frontend, backend, and data-flow interactions  
- Requires explicit owner assignments for work items and artifacts  
- Does not design or operate cloud platform, network, identity, infrastructure, governance, compliance, or data-platform architecture  
- Does not define commercial pricing, lead generation, sales packaging, product strategy, legal contract strategy, or contractual commitments  
- Does not perform live incident response, production operations execution, field support, on-call command, or direct change deployment  
- Does not issue legal, licensed, editorial, attestation, audit, release-approval, risk-acceptance, or contractual signoff

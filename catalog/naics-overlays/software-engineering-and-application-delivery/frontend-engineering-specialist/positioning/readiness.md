# Specialist Overview
The frontend engineering specialist translates design and technical requirements into actionable frontend implementation guidance and validation strategies, focusing on component architecture, rendering, accessibility, and release readiness.

## Who uses this
- Frontend engineer
- Engineering lead
- Technical reviewer
- Design-system engineer
- Quality engineer
- Delivery lead

## What it produces
- Prioritized implementation plan and artifact list
- Actionable remediation steps with verification commands
- Frontend review notes with severity, evidence, and corrective action
- Test and validation checklists with pass/fail criteria
- Modernization or migration plans with compatibility and rollback controls
- Release handoff package with packaging, telemetry, and support notes
- Orchestrator return or handoff packages with refusal rationale and residual guidance

## Risks and mitigations
- Tasks involving backend systems, cloud infrastructure, or legal requirements are explicitly returned to the orchestrator to avoid misalignment and ensure correct ownership.

## Validation
- 20 evaluation scenarios, pass rate 1
- Source coverage includes IEEE Computer Society, ISO/IEC, GOV.UK Service Manual, and ACM
- Human verification and functional audit completed

## Limitations
- What the user or organization must supply before this specialist can work reliably:
  - Clear feature definitions and UI acceptance criteria
  - Design system and component code references
  - Browser targets and accessibility goals
  - Security expectations and release constraints
- What is explicitly out of scope:
  - Backend implementation and cloud infrastructure
  - IAM and platform operations
  - Product strategy and legal approval
  - Editorial signoff and privileged deployment execution

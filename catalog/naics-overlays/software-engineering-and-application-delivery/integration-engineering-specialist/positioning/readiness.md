# Specialist Overview  
The integration engineering specialist provides guidance and review for API, webhook, file-exchange, and event-driven integrations. It transforms integration requirements into contracts, implementation guidance, validation artifacts, and operational assumptions, ensuring reliable and secure service interactions.  

## Who uses this  
- Integration engineer  
- Solution architect  
- Engineering reviewer  
- Technical delivery lead  

## What it produces  
- Contract-first implementation guidance  
- Interface review findings and remediation recommendations  
- Risk-aware design decisions for edge cases  
- Acceptance-oriented evidence records  
- Deployment/operations assumptions for integration flow  
- Review or inspection summary  
- Evidence checklist  
- Follow-up or remediation note  

## Risks and mitigations  
- Refuses to provide legal, security, or regulatory signoff. Routes such concerns to the appropriate specialist or orchestrator.  

## Validation  
- 20 evaluation scenarios, pass rate 1  
- Source coverage includes OpenAPI Initiative, AsyncAPI Initiative, IETF / RFC Editor  
- Human verification and functional audit completed  

## Limitations  
- Requires clear integration requirements and use-case context before producing guidance  
- Requires upstream design decisions on cloud platform networking, identity, and IAM to be in place  
- Requires existing security architecture reviews to be available for reference  
- Does not produce capacity planning or tuning decisions  
- Does not own legal, editorial, or regulatory signoff authority

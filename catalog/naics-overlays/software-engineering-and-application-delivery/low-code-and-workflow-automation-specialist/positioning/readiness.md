# Specialist Overview  
The low-code and workflow automation specialist provides evidence-backed guidance for the design, review, implementation, and documentation of low-code and workflow automation initiatives. This includes ensuring reliability, maintainability, security, and readiness for human-owned delivery review, while staying within defined boundaries and returning to the orchestrator when dependencies or constraints require it.

## Who uses this  
- Workflow automation builder  
- Low-code developer  
- Software delivery engineer  
- Application delivery lead  
- Automation reviewer  
- Workflow operations owner  
- Professional services delivery manager  

## What it produces  
- Workflow architecture recommendations with process stages, trigger/action design, platform fit rationale, integration dependencies, failure paths, evidence references, and assumptions  
- Implementation and hardening checklists covering connector permissions, credentials, DLP or policy checks, retries, idempotency, schema validation, tests, observability, deployment packaging, rollback, and owner signoff dependencies  
- Maintainability and correctness review notes with defect or debt observations, severity, artifact reference, recommended remediation, owner, due state, and no-regression verification  
- Troubleshooting and escalation guidance with failure classification, diagnostic evidence, replay or compensation guidance, runbook steps, incident-adjacent handoff conditions, and unresolved blockers  
- Review or inspection summary, evidence checklist, follow-up or remediation note, approval-aware client delivery note, and boundary-safe refusal or orchestrator return package  

## Validation  
- 18 evaluation scenarios, pass rate 1  
- Source coverage includes Microsoft Learn (15 instances)  
- Human verification and functional audit completed  

## Limitations  
- Requires tenant facts, source records, confidentiality constraints, approval authority, jurisdictional rules, client rights, professional judgment, or adjacent ownership to determine answers  
- Does not cover cloud platform operations, deep infrastructure hardening, site reliability execution, network configuration, IAM policy execution, core application framework implementation, legal procurement, product strategy, commercial pricing, legal advice, audit attestation, or final engineering/regulatory signoff

# Specialist Overview  
The backend engineering specialist provides technical guidance and review for backend systems, focusing on architecture, API design, implementation planning, and test strategy to help teams deliver reliable services quickly and with minimal rework or risk.

## Who uses this  
- Backend engineer  
- Technical lead  
- Code reviewer  
- Delivery reviewer  

## What it produces  
- Actionable backend implementation guidance  
- Architecture and API design notes  
- Prioritized review findings with severity and fixes  
- Test and validation matrix for functional, reliability, and security behaviors  
- Escalation and fallback recommendations for unresolved technical uncertainty  

## Risks and mitigations  
- Will not provide legal or contractual signoff; will route to appropriate authority when required  
- Will not assume privileged runtime or credential access; will document assumptions and limitations  
- Will not overstate authority in security, compliance, or regulatory decisions; will recommend escalation where necessary  

## Validation  
- 16 evaluation scenarios, pass rate 1  
- Covers backend engineering standards from IETF, OpenAPI Initiative, and NIST  
- Human verification and functional audit completed  

## Limitations  
### What the user or organization must supply before this specialist can work reliably  
- Clear requirements or design intent for the backend system  
- Access to code, API contracts, architecture notes, and test results  
- Context about existing system constraints and dependencies  

### What is explicitly out of scope  
- Frontend UI architecture, component design, or visual design systems  
- Cloud platform engineering tasks such as Kubernetes cluster operations or IAM tenancy policy  
- Product pricing, GTM strategy, or legal/commercial contract work  
- Security policy design, enterprise networking, or desktop endpoint management  
- Org-specific credentials, secrets, or privileged runtime access  
- Engineering, legal, or regulatory signoff on final decisions or releases

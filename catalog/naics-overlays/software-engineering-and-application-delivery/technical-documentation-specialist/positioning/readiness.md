# Technical Documentation Specialist

A technical documentation specialist focuses on creating, validating, and packaging implementation-oriented documentation for software engineering and application delivery. This role ensures documentation is reliable, traceable, and grounded in approved engineering evidence.

## Who uses this
- Software engineers
- Technical leads
- Engineering managers
- Technical writers
- Developer advocates
- QA reviewers
- Release coordinators
- Support enablement leads

## What it produces
- Markdown-first documentation plans or edits
- Structured review reports with remediation recommendations
- Gap analysis with assumptions and uncertainties
- Record update notes
- Document or evidence packets
- Audit trail or exception summaries
- Escalation notes for adjacent specialty work or low-confidence items

## Risks and mitigations
- Mitigates risk of documentation inaccuracy by requiring source-backed evidence for all claims.
- Rejects documentation requests where required records are missing or conflicting.
- Escalates when confidentiality, approval, or policy constraints block documentation progress.
- Avoids implying authority over engineering, legal, or operational decisions by strictly adhering to traceability and review boundaries.

## Validation
- 16 evaluation scenarios, pass rate 1.0
- Source coverage includes feature briefs, code diffs, API contracts, schema files, test evidence, architecture diagrams, runbook context, and approval status
- Human verification and functional audit completed

## Limitations
- What the user or organization must supply before this specialist can work reliably:
  - Approved engineering evidence
  - Defined documentation scope and boundaries
  - Access to documentation repositories and review matrices
  - Explicit tenant documentation policy and confidentiality rules

- What is explicitly out of scope:
  - Performing cloud platform operations or infrastructure execution
  - Implementing application code, CI/CD pipelines, or release engineering automation
  - Providing legal, compliance, or contractual approval authority
  - Owning release engineering pipelines, environment provisioning, or final release approval
  - Designing business strategy, sales collateral, or GTM narratives
  - Deciding underlying engineering design, production action, or client commitments
  - Granting editorial, security, accessibility, or publication signoff without explicit scope and reviewer assignment

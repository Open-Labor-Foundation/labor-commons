# Specialist Overview
The test automation specialist supports teams in creating and maintaining test automation that improves software quality and release confidence. They focus on strategy, design, troubleshooting, and documentation of automated testing practices across multiple layers of software systems.

## Who uses this
- Quality engineer  
- Test automation engineer  
- Software engineer  
- Engineering lead  
- Delivery reviewer  

## What it produces
- Test automation strategy matrix and implementation plan  
- Framework-specific implementation scaffolds and test patterns  
- Risk-ranked quality recommendations with measurable acceptance checks  
- Flakiness mitigation and test debt reduction playbook  
- Record update note  
- Document or evidence packet  
- Audit-trail or exception summary  
- Escalation rationale for out-of-scope or low-confidence cases  

## Risks and mitigations
- When recommending test automation for high-risk systems, the specialist routes work to a human reviewer or adjacent specialist if there is insufficient domain knowledge or clarity on responsibility boundaries.  

## Validation
- 16 evaluation scenarios, pass rate 1  
- Based on source coverage from 15 authoritative bodies and projects including ISTQB, W3C, Playwright, and Selenium  
- Human verification and functional audit completed  

## Limitations
- What the user or organization must supply:  
  - Access to testable software components and frameworks  
  - Context on feature scope and risk profiles  
  - Defined CI/CD pipeline boundaries and test execution constraints  
  - Existing test frameworks or preferred tooling stack  
- What is explicitly out of scope:  
  - Infrastructure lifecycle and cloud platform operations  
  - Production release approval or deployment promotion  
  - Legal, compliance-only, or contractual policy decisions  
  - Staffing, product roadmap, or portfolio prioritization  
  - Security risk acceptance or final attestation authority

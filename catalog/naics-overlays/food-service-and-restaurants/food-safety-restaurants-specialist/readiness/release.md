# Release Readiness

## Status

This package is `validated` as a spec pack for
`food-safety-restaurants-specialist` under the
`food-service-and-restaurants` overlay.

## Why It Is Ready

- The package defines a restaurant-specific specialty boundary grounded in food-safety controls, incident evidence, hold and release limits, and explicit refusal conditions.
- Authority sources cover workflow, terminology, evidence handling, payment-boundary handling, and failure escalation with refresh intervals and decay policies.
- Evaluation scenarios cover all required shared, workflow, and retail or hospitality abilities and meet the strict validation threshold.
- Human verification is recorded in `readiness/evidence.json`.

## Remaining Runtime Caveat

This release remains `specification-based`. It does not claim a retained implementation.
Promotion to a retained implementation requires separate runtime materialization,
retrieval wiring, usage evidence, and governance approval.

# Detection engineering specialist  
A specialist focused on improving threat detection quality through advisory and assurance activities in cybersecurity, covering detection logic design, telemetry readiness, coverage analysis, and SOC handoff.  

## Who uses this  
- Detection engineer  
- SOC engineering lead  
- Security operations manager  
- SIEM content engineer  
- EDR content engineer  
- Cloud detection engineer  
- Managed detection and response reviewer  
- Security control owner  
- Human security reviewer  

## What it produces  
- Prioritized-detection-improvement-plan  
- Validated-recommendation-memo-with-assumptions-and-confidence  
- Out-of-scope-rejection-and-escalation-recommendation  
- Readiness-check-for-deployment-and-commercialization  
- Quality-or-safety-review-note  
- Control-or-incident-summary  
- Corrective-action-escalation  
- Telemetry-prerequisite-and-evidence-gap-checklist  
- Hold-release-or-rollback-recommendation  
- Source-or-policy-conflict-note  

## Risks and mitigations  
- When telemetry, policy, or source conflicts prevent reliable detection guidance, the specialist refuses to proceed and escalates the issue for resolution.  

## Validation  
- 20 evaluation scenarios, pass rate 1  
- Source coverage includes NIST (11 references)  
- Human verification and functional audit completed  

## Limitations  
- What the user or organization must supply before this specialist can work reliably:  
  - Detection-engineering requests with clear hypotheses or requirements  
  - Access to telemetry sources and system definitions  
  - Existing detection rules, logic, or coverage baselines  
  - Defined ATT&CK or control mapping context  

- What is explicitly out of scope:  
  - Performing penetration-testing execution, exploitation, red-team reenactment, or adversary emulation  
  - Executing incident response containment, recovery, or SOAR playbook actions  
  - Issuing formal audit attestation, compliance certification, or breach notification  
  - Owning broad enterprise architecture, cloud migration, or identity platform design  
  - Performing malware reverse engineering, forensic acquisition, or deep endpoint analysis  
  - Treating cybersecurity or detection engineering as cosmetic or un-evidenced labels

# Specialist Overview
The security architecture specialist translates architecture evidence, identity and trust context, control baselines, threat inputs, and resilience constraints into scoped security architecture assessments, trust-boundary designs, control-to-risk rationale, and prioritized remediation plans.

## Who uses this
- Security architect
- Platform architect
- Cloud architect
- Application owner
- Security reviewer
- Risk or compliance stakeholder

## What it produces
- Security architecture assessment note
- Control-to-risk rationale matrix
- Segmentation and trust boundary design proposal
- Prioritized remediation roadmap
- Escalation memo when confidence or evidence is insufficient

## Validation
- 16 evaluation scenarios, pass rate 1
- Source coverage includes National Institute of Standards and Technology (NIST), Cybersecurity and Infrastructure Security Agency (CISA), and other authoritative cybersecurity frameworks
- Human verification and functional audit completed

## Limitations
- What the user or organization must supply before this specialist can work reliably
  - Architecture artifacts and context
  - Identity and trust boundaries
  - Threat model inputs
  - Resilience constraints
  - Control baseline references
- What is explicitly out of scope
  - Penetration testing, scanning, exploit validation, or vulnerability proof-of-concept work
  - Incident response execution, containment, eradication, recovery, or forensics
  - Formal third-party audit opinions, certification conclusions, regulatory attestation, or legally binding compliance wording
  - Enterprise application architecture, product, cost, performance, or platform design decisions outside security scope
  - Rewriting policy, governance, legal, procurement, or contractual artifacts when implementation guidance is sufficient
  - Approving production changes, risk acceptance, exceptions, or control waivers

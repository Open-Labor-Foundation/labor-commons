# Specialist Overview  
The network security specialist provides advisory guidance on network-layer security, reviewing configurations, policies, and telemetry to produce risk-ranked recommendations, remediation sequences, and evidence-based exception guidance, while staying within the boundaries of network controls and operations.

## Who uses this  
- Network security engineer  
- Network architect  
- Security architect  
- SOC analyst  
- Security operations lead  
- Change advisory reviewer  
- GRC evidence owner  
- Human reviewer  

## What it produces  
- Risk-ranked network security assessment with evidence citations  
- Remediation sequence with sequence and impact rationale  
- Schedule or allocation recommendation  
- Constraint log  
- Clear in-scope/out-of-scope decision and escalation note  
- Evidence-backed exception and exception-risk guidance  
- Source-or-policy conflict note  
- Adjacent-specialist handoff package  

## Risks and mitigations  
- Refuses to generate or recommend execution of penetration tests, exploit development, attack simulation, or incident response actions to avoid misattribution of authority.  
- Refuses to provide legal, compliance, or final approval statements to maintain alignment with advisory and evidence-bound lane.  
- Mitigates scope creep by explicitly routing non-network-related security tasks to adjacent specialists such as identity, cloud, or endpoint security.  

## Validation  
- 18 evaluation scenarios, pass rate 1  
- Source coverage includes NIST, CISA, and other authoritative network security frameworks  
- Human verification and functional audit completed  

## Limitations  
- Requires accurate and up-to-date network topology, policy, and telemetry artifacts before producing recommendations  
- Requires clear escalation paths and adjacent specialist availability for handoffs  
- Requires human review of all final decisions and recommendations  
- What is explicitly out of scope  
  - Penetration testing execution, exploit development, adversary emulation, or attack simulation  
  - Incident response containment, forensics, recovery, or production network changes  
  - Broad enterprise architecture ownership outside network controls and operations  
  - IAM policy, endpoint configuration, cloud platform implementation, or application remediation outside direct network relevance  
  - Legal, regulatory, compliance, certification, audit attestation, risk acceptance, or final approval authority  
  - Budget, staffing, or production change approvals  
  - Cybersecurity as a cosmetic or marketing label

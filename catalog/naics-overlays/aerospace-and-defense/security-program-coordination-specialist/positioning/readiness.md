# Specialist Overview  
The security program coordination specialist manages and coordinates security program workflow dependencies in the aerospace-and-defense industry. It ensures protective controls, security incidents, supplier events, and traceability dependencies are tracked and routed with explicit ownership and clear handoffs across security, quality, production, and logistics teams, without assuming direct authority over legal, safety-critical, or release decisions.

## Who uses this  
- Security program coordinator  
- Security operations coordinator  
- Security engineer  
- Quality systems security liaison  
- Production planner  
- Inventory operations reviewer  
- Supplier quality security reviewer  
- Logistics security coordinator  

## What it produces  
- Handoff-summary  
- Dependency-tracker  
- Routing-confirmation-note  
- Handoff-status-summary  
- Blocked-dependency-escalation-note  
- Closure-confirmation  
- Traceability-conflict-summary  
- Quality-or-release-boundary-escalation  
- Supplier-security-or-nonconformance-escalation  
- Security-program-routing-package  

## Risks and mitigations  
- Direct execution of security control changes or production actions is explicitly refused to avoid unauthorized authority use.  
- Escalations are required for legal, safety-critical, or regulated authority decisions to ensure proper human review.  
- Refusals are issued for incomplete tenant facts or missing authoritative records to prevent unsafe or noncompliant progress.  

## Validation  
- 12 evaluation scenarios, pass rate 1  
- Source coverage includes 8 authority sources: Electronic Code of Federal Regulations, Federal Aviation Administration, International Aerospace Quality Group, and related aerospace quality standards  
- Human verification and functional audit completed  

## Limitations  
- What the user or organization must supply before this specialist can work reliably:  
  - Minimum tenant facts  
  - Authoritative records  
  - Source context  
  - Release/safety boundaries  

- What is explicitly out of scope:  
  - Approving legal, licensing, policy-override, financial, clinical, safety-critical, or regulated compliance authority decisions  
  - Executing security control implementation, direct vulnerability remediation, supplier substitution, or production execution actions  
  - Releasing held or nonconforming product, performing airworthiness signoff, closing MRB cases, or providing final quality release authority  
  - Bypassing traceability, hold, release, supplier-control, or security incident handling controls to preserve schedule or throughput

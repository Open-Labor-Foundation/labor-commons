# Specialist Overview
A data integration specialist focuses on system-to-system data movement for analytics and AI delivery. This includes selecting ingestion patterns, advising on connectors and change data capture (CDC), planning replication cutover, managing schema compatibility, handling integration runtime operations, and addressing failure and replay scenarios for data between operational sources and analytical systems.

## Who uses this
- Data engineer
- Integration engineer
- Analytics platform engineer
- Reviewer

## What it produces
- Integration architecture recommendation with assumptions and tradeoffs
- Connector or CDC design notes with quality and recovery controls
- Cutover, replay, and rollback plan for data movement changes
- Boundary-safe rejection or escalation response
- Evidence-linked recommendation summary with cited authority sources

## Validation
- 12 evaluation scenarios, pass rate 1
- Source coverage includes Amazon Web Services and Microsoft Learn
- Human verification and functional audit completed

## Limitations
- Requires clear definitions of source and target systems, data formats, and integration goals before work begins
- Cannot handle tasks involving enterprise data platform blueprints, business intelligence report design, general software integration engineering, or unrelated infrastructure operations

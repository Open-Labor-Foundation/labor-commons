# Release Notes

## v1.1.0

Refreshed the VPN and remote access specialist into a validated spec pack for
issue #1657.

## Release Scope

- Reframed the package as the canonical spec-first definition with
  `runtime_strategy: specification-based`.
- Added intake, eligibility, authorization, prerequisite validation, queue
  states, output shapes, traceability expectations, and materialization
  requirements.
- Added `evaluation/research-summary.json` with research contract version
  `2026-04-08`, cross-industry source classes, workflow stages, artifacts,
  systems of record, decision boundaries, failure modes, scenario family
  coverage, and unresolved ambiguity handling.
- Added `evaluation/functionality-map.json` covering all shared abilities,
  intake/access workflow abilities, and cross-industry artifact awareness.
- Expanded evaluation from 12 legacy scenarios to 18 refreshed scenarios.
- Updated deployment, rollback, tenant isolation, monitoring, commercialization,
  and promotion criteria.

## Validation

- Validation profile: standard.
- Scenario count: 18.
- Pass count: 18.
- Pass rate: 1.0.
- Minimum pass rate: 0.90.
- Human verification: recorded.
- Functionality coverage: complete.
- Unreviewed regressions: none.

## Runtime Posture

The package remains `specification-based`. It does not claim retained implementation
ownership and does not justify long-term storage of a generated implementation.
A generated implementation can be promoted only after frequent-use evidence,
scenario stability, tenant isolation, rollback success, and repository owner
approval are recorded.

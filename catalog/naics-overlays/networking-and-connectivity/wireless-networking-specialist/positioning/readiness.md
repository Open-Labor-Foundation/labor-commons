# Specialist Overview  
The wireless networking specialist delivers source-grounded guidance for wireless network architecture, operations, and troubleshooting in campus, branch, and remote-office environments. They focus on advisory and specification-first work, producing actionable recommendations and escalation notes without executing changes or making regulatory or procurement decisions.

## Who uses this  
- Network architect  
- Wireless engineer  
- Network operations engineer  
- Platform networking owner  
- Managed network services reviewer  
- Site reliability or service desk escalation owner  

## What it produces  
- Intake-disposition-note  
- Eligibility-or-prerequisite-checklist  
- Routing-or-escalation-summary  
- Actionable-auditable-wireless-design-guidance-with-configuration-checklist-and-tradeoff-rationale  
- Troubleshooting-plan-with-isolation-steps-and-success-criteria  
- Boundary-safe-routing-addressing-recommendation-for-client-and-guest-traffic-paths  
- Deployment-and-commercialization-evidence-note-aligned-to-contract-gates  
- Source-or-policy-conflict-note  
- Low-confidence-escalation-note-with-missing-facts  

## Risks and mitigations  
- Refuses to act on requests involving spectrum or regulatory compliance, legal decisions, or safety-critical operations.  
- Routes identity, firewall, WAN, or endpoint work to appropriate specialists to avoid overstepping authority.  
- Escalates low-confidence scenarios with missing facts to ensure decisions are made by qualified personnel.  

## Validation  
- 18 evaluation scenarios, pass rate 1  
- Source coverage includes NIST, IEEE, IETF, and IETF  
- Human verification and functional audit completed  

## Limitations  
- What the user or organization must supply before this specialist can work reliably  
  - Wireless estate topology and device inventory  
  - Current SSID and VLAN mapping  
  - Existing RF and channel usage reports  
  - DHCP/DNS/IPv6 configuration status  
  - Wireless client and access point logs when troubleshooting  

- What is explicitly out of scope  
  - Endpoint device management or OS hardening  
  - Identity/IAM platform design beyond WLAN integration  
  - Application code or API debugging  
  - Direct vendor command scripting or privileged changes outside wireless operations  
  - Regulatory, legal, or procurement decisions  
  - Broad networking mandates covering firewalls, WAN, BGP, or cloud architecture when wireless is not primary concern

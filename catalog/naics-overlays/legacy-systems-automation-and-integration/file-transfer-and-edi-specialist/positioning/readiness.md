# Specialist Overview  
The File transfer and EDI specialist focuses on legacy and hybrid-business integration tasks involving managed file transfer, EDI protocol design, trading-partner onboarding, and operational troubleshooting. It provides clear guidance for protocol selection, secure exchange setup, certificate management, and modernization of file-transfer and EDI estates, while strictly enforcing boundaries to avoid adjacent integration or software work.

## Who uses this  
- edi-analyst  
- b2b-integration-engineer  
- managed-file-transfer-admin  
- legacy-modernization-lead  
- reviewer  

## What it produces  
- Protocol and platform recommendation with assumptions, constraints, and cited sources  
- Partner onboarding or remediation checklist  
- Failure-analysis or replay runbook  
- Migration and rollback plan for file-transfer or EDI workloads  
- Boundary-safe rejection or escalation note when the request leaves this specialty  

## Risks and mitigations  
- The specialist explicitly refuses to engage in tasks involving legal interpretation of contracts, regulatory obligations, or X12 licensing terms. It also declines to handle live security operations, custom software development, or cloud architecture outside its defined scope.  

## Validation  
- 12 evaluation scenarios, pass rate 1  
- Covers AS2, SFTP, FTPS, X12, and EDIFACT standards, with source material from IETF, X12, and UNECE  
- Human verification and functional audit completed  

## Limitations  
- What the user or organization must supply before this specialist can work reliably  
  - Clear scope and boundaries of the integration or migration task  
  - Specific file-transfer or EDI protocol requirements  
  - Known trading-partner profiles and communication preferences  
  - Existing infrastructure details for legacy systems (e.g., BizTalk, IBM Sterling)  
- What is explicitly out of scope  
  - Broad middleware integration, iPaaS, API, or event-driven architecture design  
  - Custom software development, ERP mapping, or application logic beyond EDI boundaries  
  - Cloud architecture, firewall design, or job-scheduling tasks  
  - Legal interpretation of contracts or X12 licensing  
  - Security incident response, penetration testing, or digital forensics

# Specialist Overview  
The middleware integration specialist focuses on reliable message exchange and interoperability in legacy and hybrid estates, providing evidence-based guidance on protocol mediation, delivery guarantees, failure handling, and cutover planning between established middleware platforms.  

## Who uses this  
- Integration architect  
- Middleware engineer  
- Legacy modernization lead  
- Application support engineer  
- Enterprise architect  

## What it produces  
- Middleware integration decision record with recommended topology, delivery guarantees, and rationale  
- Failure-mode remediation guidance covering retry, DLQ or backout handling, correlation, and observability  
- Coexistence or migration wave plan with cutover risks, rollback path, and escalation checkpoints  
- Boundary-safe refusal or low-confidence escalation memo  
- Structured handoff notes to adjacent specialists when work is primarily API, event, EDI, file transfer, batch, RPA, or broader modernization planning  

## Validation  
- 12 evaluation scenarios, pass rate 1  
- Guidance based on IBM and Oracle middleware practices and integration patterns  
- Human verification and functional audit completed  

## Limitations  
- Requires a clear legacy or hybrid middleware context with defined interfaces and constraints  
- Requires availability of source schema, queue policies, and transaction ownership details to operate reliably  
- Does not own broad greenfield integration platform selection  
- Does not own API design, event-streaming strategy, or managed file transfer operations  
- Does not own batch workflow design, RPA automation, or full modernization-program sequencing

# Specialist Overview  
The ERP platform specialist provides guidance for configuring, administering, and operating enterprise resource planning (ERP) systems. It focuses on platform-specific decisions for ERP suites such as Dynamics 365, Oracle Fusion Cloud ERP, and SAP S/4HANA Cloud, including integration planning, role governance, release readiness, and troubleshooting.

## Who uses this  
- ERP platform owner  
- Application administrator  
- Enterprise architect reviewer  
- Systems integrator  
- Operations lead  

## What it produces  
- ERP platform action plan with assumptions, sequence, validation points, and rollback path  
- Boundary-safe approval, workflow, role, or integration recommendations  
- Release and operations readiness review with risk statements and escalation points  
- Out-of-scope refusal or partial handoff note for adjacent specialties  
- Evidence-ready notes aligned to validated repository contract requirements  

## Validation  
- 12 evaluation scenarios, pass rate 1  
- Source coverage includes Microsoft Learn  
- Human verification and functional audit completed  

## Limitations  
- User must supply a specific ERP platform context and evidence of current tenant setup, module usage, and integration touchpoints  
- User must clarify the scope of the request to ensure it is platform-scoped  
- User must provide details about current workflows, role definitions, and governance patterns if they are part of the request  

- Custom ERP extensions, application code, middleware code, or CI/CD pipelines  
- CRM-only, HCM-only, payroll-only, or non-ERP enterprise application strategy  
- Accounting policy, tax, legal, or regulatory attestation  
- Cloud landing zones, networking, identity platforms, endpoint fleets, or unrelated infrastructure operations  
- Vendor support replacement, licensed implementation partner work, or tenant-specific legal approval authority

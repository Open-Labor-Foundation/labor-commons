# Specialist Overview  
The enterprise application integration specialist supports safe, evidence-based integration decisions between packaged enterprise applications using vendor-supported integration capabilities. This specialist focuses on pattern selection, configuration guidance, troubleshooting, and governance for app-to-app integrations, and explicitly rejects work outside this lane such as custom development or full platform ownership.

## Who uses this  
- Enterprise applications lead  
- Integration architect  
- Application administrator  
- iPaaS administrator  
- Operations reviewer  
- Reviewer  

## What it produces  
- Integration design note with supported pattern choice, assumptions, sequencing, and rollback path  
- Mapping and data-ownership review with validation steps and risk statements  
- Operations or troubleshooting memo with monitoring checks, error-triage order, and escalation boundaries  
- Governance recommendation covering connector controls, tenant isolation, and source-backed limitations  
- Clear refusal and routing note for out-of-scope work  

## Validation  
- 13 evaluation scenarios, pass rate 1  
- Source coverage: Microsoft Learn (13 times)  
- Human verification and functional audit completed  

## Limitations  
- Requires access to vendor-published integration documentation and supported APIs  
- Requires clear problem statement with specific integration use case and target application systems  
- Requires access to existing integration artifacts or baseline configurations for review or troubleshooting  

- Custom middleware implementation  
- Bespoke software engineering  
- Tenant-wide identity architecture  
- Master data governance ownership  
- Warehouse or ETL design  
- Unrelated infrastructure administration  
- Full platform ownership for source or target application outside the integration lane

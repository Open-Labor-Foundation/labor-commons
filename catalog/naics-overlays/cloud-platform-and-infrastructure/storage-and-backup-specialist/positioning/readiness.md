# Specialist Overview  
The storage and backup specialist provides practical, actionable guidance for building and maintaining storage and backup programs in cloud infrastructure. This includes designing data-risk-aware systems, defining recovery strategies, aligning with policies, and creating operational playbooks to ensure reliable restores and defensible recovery evidence.

## Who uses this  
- Cloud platform engineers  
- Infrastructure reliability leads  
- Backup program owners  
- Operations reviewers  

## What it produces  
- Strategy matrix  
- Backup and retention architecture  
- Runbook and escalation guidance  
- Compliance-aligned design evidence  
- Cost and risk tradeoff recommendations  
- Record update note  
- Document or evidence packet  
- Audit-trail or exception summary  

## Risks and mitigations  
- When generating guidance for high-availability or mission-critical systems, the specialist explicitly refuses to make decisions about live systems, regulatory compliance, or production data handling without human validation.  

## Validation  
- 20 evaluation scenarios, pass rate 1  
- Source coverage includes four Amazon Web Services references  
- Human verification and functional audit completed  

## Limitations  
- **What the user or organization must supply:**  
  - Detailed workload classification and data sensitivity  
  - Existing infrastructure topology and storage usage  
  - Compliance or regulatory requirements  
  - Budgetary constraints for storage and backup tiers  

- **What is explicitly out of scope:**  
  - Vendor-specific configuration scripts or automation code  
  - Real-time monitoring or alerting system design  
  - Direct implementation of backup tools or services  
  - Forensic analysis of actual data breaches or ransomware attacks  
  - Legal or compliance certification of backup systems

# Specialist Overview  
The Cloud landing zone specialist focuses on defining and governing cloud infrastructure foundations across Azure, AWS, and Google Cloud. It delivers structured guidance on landing-zone architecture, control baselines, IaC bootstrap, and operational guardrails while maintaining clear boundaries on what is in or out of scope.

## Who uses this  
- Platform-architect  
- Cloud-foundation-lead  
- Cloud-operations-engineer  
- Infrastructure-delivery-manager  
- Architecture-reviewer  

## What it produces  
- Implementation-ready landing-zone blueprint  
- Control-and-guardrail recommendation matrix  
- IaC module or module-family selection and structure guidance  
- Operations runbooks and incident response checks  
- Handoff summary  
- Dependency or blocker log  
- Routing confirmation note  
- Out-of-scope rejection rationale and escalation notes  

## Validation  
- 18 evaluation scenarios, pass rate 1.0  
- Source coverage limited to Microsoft Learn (17 sources)  
- Human verification and functional audit completed  

## Limitations  
- The user or organization must supply: cloud provider preference, organizational structure, compliance constraints, and existing infrastructure constraints.  
- Explicitly out of scope: application engineering, live tenant execution, legal/compliance signoff, deep security lawyering, MLOps, data architecture, vendor support operations, DNS/application migration execution, and non-cloud infrastructure specialties.

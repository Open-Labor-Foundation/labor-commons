# Specialist Overview  
The platform engineering specialist provides advisory and artifact production for shared cloud and infrastructure platforms, focusing on operating model design, landing-zone architecture, platform guardrails, infrastructure-as-code workflows, platform reliability, runbooks, and auditable records. It operates within a clear boundary of infrastructure platform concerns and rejects tasks outside its domain.  

## Who uses this  
- Platform engineers  
- Cloud platform leads  
- Site reliability engineers (SRE)  
- Cloud operations leads  
- Solution architects  
- Architecture reviewers  

## What it produces  
- Platform architecture and reference design recommendations  
- Implementation and enablement roadmap  
- Failure-mode and recovery playbooks  
- Guardrail and policy proposals with escalation matrix  
- Cost/availability/reliability rationale  
- Record update note  
- Document or evidence packet  
- Audit-trail or exception summary  
- Out-of-scope rejection rationale and orchestrator return note  

## Validation  
- 18 evaluation scenarios, pass rate 1.0  
- Covers authoritative sources including CNCF TAG App Delivery Platforms Working Group, Microsoft Learn, and Amazon Web Services  
- Human verification and functional audit completed  

## Limitations  
- Requires the user to supply context about existing infrastructure, platform operating models, and compliance requirements before producing reliable outputs  
- Explicitly out of scope:  
  - Application-only engineering tasks  
  - Security specialties unrelated to infrastructure platforms  
  - Endpoint security  
  - Legal, procurement, or staffing authority decisions  
  - Pure FinOps approval workflows  
  - Non-cloud infrastructure specialties

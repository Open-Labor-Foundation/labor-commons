# Specialist Overview  
The Database Platform Operations Specialist provides advisory and execution-support guidance for database operations in cloud environments. It focuses on platform-level decisions for managed and self-managed database services, including architecture, deployment, resilience, backup, recovery, observability, and maintenance guardrails, while maintaining clear operational boundaries.

## Who uses this  
- Database platform engineers  
- Cloud platform engineers  
- Site reliability engineers  
- Infrastructure architects  
- Technical reviewers and change owners  

## What it produces  
- Shortlist of cloud-native database platform patterns with tradeoffs  
- Resiliency and failover architecture decision record  
- Backup, restore, and failover runbooks  
- IaC and configuration checkpoints for deployment  
- Incident response checklist mapped to severity  
- Execution or case disposition summary  
- Exception queue note  
- Completion evidence record  
- Explicit in-scope versus out-of-scope boundary statements  

## Risks and mitigations  
The specialist does not claim ownership over application queries, security operations, legal compliance, or production execution. It routes work to adjacent specialists or human reviewers when ownership shifts lanes, ensuring no overreach into high-stakes or regulated domains.

## Validation  
- 18 evaluation scenarios, pass rate 1  
- Covers Amazon Web Services and Microsoft Learn sources  
- Human verification and functional audit completed  

## Limitations  
- Requires workload profile, cloud environment, and current platform state as inputs  
- Does not own application schema design, security policy creation, procurement, legal interpretation, or final production execution  
- Does not support non-cloud platforms or executive-level decisions  
- Cannot provide exact pricing quotes, breach response, or compliance attestation

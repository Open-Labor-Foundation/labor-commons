# Specialist Overview  
The Kubernetes and container platform specialist focuses on infrastructure-level Kubernetes and container platform design, implementation guidance, operations, and troubleshooting, with a clear boundary on infrastructure decisions and platform guardrails.

## Who uses this  
- Platform architect  
- Platform engineer  
- Site reliability engineer  
- Cloud operations engineer  
- Architecture reviewer  

## What it produces  
- Architecture recommendations with tradeoff rationale  
- Implementation blueprints and platform guardrails  
- Incident triage and remediation plans  
- Readiness checklists and release-evidence summaries  
- Schedule or allocation recommendations  
- Constraint logs  
- Escalation notes for approval-required tradeoffs  
- Decision logs for out-of-scope and escalation routing  

## Validation  
- 18 evaluation scenarios, pass rate 1.0  
- Source coverage includes Kubernetes Documentation Project (21 references)  
- Human verification and functional audit completed  

## Limitations  
- Requires user or organization to supply cluster context, workload profiles, and provider constraints  
- Explicitly out of scope:  
  - Application-only engineering decisions  
  - Unrelated security specialties  
  - Workplace technology operations  
  - Procurement negotiations  
  - Non-cloud specialties  
  - Work requiring final approval, legal/compliance signoff, credentialed tenant execution, or provider support ownership outside infrastructure lane
